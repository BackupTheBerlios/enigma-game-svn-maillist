+<h3><a class="arrowup" href="$$development$$"><img src="$$imagedir$$/nav_up.gif" border="0" title="back to top">
+  <a name="tasks">Open Tasks and Whishes</a></h3>
+
+  <p>Here is a list of open tasks, most of which we currently don't have 
+  the resources to tackle ourselves.  We <i>definitely need</i> your help
+  with these.  If you have the necessary skills and some spare time,
+  please consider helping us out. </p>
+
+  <ul>
+  <li><b>Translation</b>.  Enigma has proper support for languages other than
+      English.  If you would like to help us localize Enigma to your native
+      language, please contact us.  This concerns not only the buttons and
+      levels in Enigma, but also the manuals, the FAQ, the Wiki and this
+      homepage.  (Translation into different dialects are interesting,
+      too!) <i>Easy</i></li>
+      
+  <li><b>Better graphics</b>.  We're currently lacking in the arts
+      department, as you may have noticed.  If you have more talent than we
+      have, we'd be only too glad to replace our sprites, logos, and other
+      images with better ones. <i>Easy-Medium</i></li>
+
+  <li><b>Network game</b>.  This is a big and non-trivial undertaking.
+      If you like challenges and have some experience in network
+      programming, feel free to join us. <i>Difficult</i></li>
+
+  <li><b>Level editor</b>.  This has been requested by many
+      users, and would in fact be a great addition.  It's a good deal
+      of work, but reasonably easy.  Use of Java seems sensible, however, you
+      could as well work with the existing editors and update
+      them.  (Well, Mac-users would like to have an editor,
+      too.) <i>Medium</i></li>
+      
+  <li><b>Levels</b>.  We're constantly searching (good) levels.  Although
+      there currently is no editor for 1.00 available, programming levels
+      isn't too difficult, given the existing tools. <i>Easy-Medium</i></li>
+
+  <li><b>Sounds</b>.  Sounds are currently incomplete, and some of the
+      existing ones might (do!) need some polish. <i>Medium</i></li>
+
+  <li><b>Ports</b>.  Although Enigma already covers most important platforms,
+      there are still many open points: More packs for the different
+      Linux-distributions, tests to compile Enigma with Visual Studio, another
+      active programmer to help us fix bugs that occur on Macs, exotic targets
+      like game consoles or handhelds. <i>Easy-Difficult</i></li>
+
+  <li><b>Maintaining the homepage</b>.  Someone has to add news and monthly
+      updates (like ratings user statistics) to the homepage.  This is not
+      much work, dependent on what is planned for the homepage.  But
+      nevertheless we need someone who does this regularly for
+      us. <i>Easy</i></li>
+
+  <li><b>Tech-support</b>.  The larger Enigmas code gets, the more technical
+      questions and problems arise on mag-heut.net and devel-ml.  This
+      sometimes involves the monthly repetition of the ever same answers,
+      sometimes really nasty bug-chases, and sometimes just the redirection of
+      the question to the right people. We'd be glad if someone who's also
+      able to dive into foreign code could help us with this
+      task. <i>Easy-Medium</i></li>
+
+  </ul>

+<h3><a class="arrowup" href="$$development$$"><img src="$$imagedir$$/nav_up.gif" border="0" title="back to top">
+  <a name="level">Level Development</a></h3>
+
+  <p>Even though Enigma does not include a level editor and you may have to dig
+  into the simple script language Lua developing a level is easier than you may
+  assume.</p>
+  
+  <p>The Enigma <a href="$$refman$$">reference manual</a> explains the level 
+  format and all objects you can use in detail. You should at least cross read 
+  chapters 1 - 3 and remember that you can look up details about every object.<p>
+  
+  <p>For your first experiments and as a practical guide to level coding you 
+  should have a look at the <a href="$$wiki$$">level development wiki</a>. It 
+  focuses on the most common object features and gives you valuable code 
+  snippets for many common tasks.</p>
+  
+  <p>You will find help on the <a href="$$development$$#maillist">developer 
+  mailing list</a> as well as in the <a href="http://www.mag-heut.net">forum</a>.
+  As soon as you think that your level is finished, set the level XML
+  attribut 'status' to 'stable' and publish the level to
+  the mailing list and/or to the forum. People will test and comment your level.
+  When no shortcuts are found and you finish the development of a level you
+  should set the status to 'released' and republish this version.
+  We collect the levels from these sources and will contact you on remaining
+  issues.</p>
+

-<h3><a class="arrowup" href="$$support$$"><img src="$$imagedir$$/nav_up.gif" border="0" title="back to top">
-  <a name="score">Score and Rating Registration</a></h3>
-
-  <p>Starting with Enigma 1.00 you can register your scores and ratings. Why is
-  it useful to submit your results? Based on as many user results as possible
-  we calculate statistics that you use by inspecting a levels info. Thus even
-  the fact that many users did not solve certain levels is as important as the
-  knowledge of top scores for improving Enigma.</p>
-  
-  <p>Please make use of the possibility to rate levels. Right or ctrl click on
-  the level icon in the level menu to view the level inspector. At the bottom
-  line you can rate a level with values from 0 (bad) to 10 (top). Try to judge
-  bad levels as well as good levels. Average levels should be judged by a value
-  of 5. Your judgement shows which levels users like and is the basis for the
-  'Level of the Month' selection.</p>
-
-  <p>You need to choose a unique user name in the options menu. Please look at
-  the <a href="$$statistics$$#userlist">users list</a> if your favorite name
-  is not in use by someone else.</p>
-
-  <p>For registration of your scores just send us your 'enigma.score'
-  file. It is located at your 'user path' that Enigma shows you on
-  pressing F2 at the main menu. Please send just this one file to
-  <a href="mailto:score at enigma-game.org">score at enigma-game.org</a>
-  with subject 'score - username' (substitute 'username' by
-  the name you have chosen!).</p>
-
-  <p> The scores are evaluated on a monthly base. Please do not send your score file
-  after every good score that you achieved. Once at the end of the month is
-  sufficient.</p> 
-
-  <p>The updated statistics will be downloaded to your computer if you set the
-  ratings update option to 'Auto'. Enigma will peek the server only a few times
-  per month, independent how often you start the application. The network traffic
-  is very low as just increments are downloaded and you will receive all updates
-  within a few days.</p>
-
-  <p>If you play most times offline and set the update option to 'Never'
-  you may manually download the ratings update file
-  <a href="http://www.nongnu.org/enigma/ratings/ratings.xml.zip">ratings.xml</a>.
-  Extract the file from the zip and copy it to your 'user path' aside

+<h3><a class="arrowup" href="$$support$$"><img src="$$imagedir$$/nav_up.gif" border="0" title="back to top">
+  <a name="score">Score and Rating Registration</a></h3>
+
+  <p>Starting with Enigma 1.00 you can register your scores and ratings. Why is
+  it useful to submit your results? Based on as many user results as possible
+  we calculate statistics that you use by inspecting a levels info. Thus even
+  the fact that many users did not solve certain levels is as important as the
+  knowledge of top scores for improving Enigma.</p>
+  
+  <p>Please make use of the possibility to rate levels. Right or ctrl click on
+  the level icon in the level menu to view the level inspector. At the bottom
+  line you can rate a level with values from 0 (bad) to 10 (top). Try to judge
+  bad levels as well as good levels. Average levels should be judged by a value
+  of 5. Your judgement shows which levels users like and is the basis for the
+  'Level of the Month' selection.</p>
+
+  <p>You need to choose a unique user name in the options menu. Please look at
+  the <a href="$$statistics$$#userlist">users list</a> if your favorite name
+  is not in use by someone else.</p>
+
+  <p>For registration of your scores just send us your 'enigma.score'
+  file. It is located at your 'user path' that Enigma shows you on
+  pressing F2 at the main menu. Please send just this one file to
+  <a href="mailto:score at enigma-game.org">score at enigma-game.org</a>
+  with subject 'score - username' (substitute 'username' by
+  the name you have chosen!).</p>
+
+  <p> The scores are evaluated on a monthly base. Please do not send your score file
+  after every good score that you achieved. Once at the end of the month is
+  sufficient.</p> 
+
+  <p>The updated statistics will be downloaded to your computer if you set the
+  ratings update option to 'Auto'. Enigma will peek the server only a few times
+  per month, independent how often you start the application. The network traffic
+  is very low as just increments are downloaded and you will receive all updates
+  within a few days.</p>
+
+  <p>If you play most times offline and set the update option to 'Never'
+  you may manually download the ratings update file
+  <a href="http://www.nongnu.org/enigma/ratings/ratings.xml.zip">ratings.xml</a>.
+  Extract the file from the zip and copy it to your 'user path' aside
+  your 'enigma.score' file.</p>

+<h3><a class="arrowup" href="$$support$$"><img src="$$imagedir$$/nav_up.gif" border="0" title="back to top">
+  <a name="bug">Bug Reporting</a></h3>
+
+  <p>We appreciate any report about problems with Enigma. Of course a software
+  with more than 200 000 lines of code and complex gaming logic like Enigma is
+  never bug free. But Enigma reached a quite stable 1.00 release with many
+  thousands of users. Please first look at the documentation, faq and user
+  forum for possible similar reports.</p>
+  
+  <p>For problems besides the solvability of levels send a mail to the developers
+  mailing list <a href="mailto:enigma-devel at nongnu.org" >enigma-devel at nongnu.org</a>
+  (you do not need to subscribe to the list).
+  Please include the following basic information:
+    <ul>
+    <li>What operating system are you using?</li>
+    <li>Which Enigma version are you using?</li>
+    <li>What did you do when the bug happened?</li>
+    <li>What exactly did you see then?</li>
+    <li>Is the problem ongoing or reproducable?</li>
+    </ul>
+  If you file a report please stay tuned on and answer further requests on
+  specific details by our developers. Do not expect immediate answers - it may
+  take some time for the right people to answer your request.</p>
+  
+  <p>Problems with single levels should be reported to the user forum 
+  <a href="http://www.mag-heut.net">mag-heut.net</a>. Please keep in mind that 
+  all levels have been proven solvable with Enigma 1.00. But unintended shortcuts 
+  or annoyances with levels will remain an ongoing topic and may sometimes emerge
+  to be caused by inconsistencies within Enigma itself.</p>
+  
+  <p>Developers may want to file bug reports directly to our bug database at 
+  <a href="http://developer.berlios.de/projects/enigma-game/">Berlios.</a>
